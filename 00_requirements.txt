# **FridgeMeal(**냉장고 기반 메뉴 추천 서비스)

## 1. 프로젝트 개요 (Overview)

- **프로젝트명** : 냉장고 기반 메뉴 추천 서비스 - **FridgeMeal**
- **프로젝트 소개**
    - **소개** : 냉장고에 있는 재료를 기반으로 오늘 해먹을 메뉴를 추천해주는 서비스. 사용자는 현재 보유한 식재료를 활용해 만들 수 있는 메뉴를 쉽고 빠르게 확인할 수 있으며, 메뉴를 결정하는 시간을 줄이고, 보유한 식재료를 더욱 효율적으로 활용할 수 있도록 돕는다.
    - **서비스 컨셉** : "오늘 뭐 해먹지?"를 가장 적은 고민으로 해결하는 서비스
    - **서비스 핵심 가치** : 보유한 식재료를 효율적으로 활용해 메뉴 결정을 돕는다.
- **타겟 사용자** : 냉장고 속 재료를 활용해 집에서 요리하는 사람
- **문제 현상 및 근거** :
    - 메뉴를 결정하는 데 어려움을 겪는다.
        - Meal prepping survey (UX 연구) 조사에서 “새로운 아이디어 고안”과 “메뉴 다양성 계획”이 가장 어려운 요인 2개로 꼽혔고, 응답자 100%가 기존 meal planning 앱을 사용하지 않는다고 답했다. https://medium.com/@PatriciaNamara_/mealmate-78ad3d5058f
        - Allspice Labs 연구에서는 online recipe 사용 행동에서 “무엇을 요리할지 정하는 것”이 큰 부담 중 하나이며, 레시피 사이트/앱 사용의 주요 동기가 “요리 아이디어 부족”과 “메뉴 고민 시간 단축”으로 나타난다. https://www.allspicelabs.com/data/study/
        
        ⇒ 많은 사람들은 '오늘 무엇을 해먹을지'를 결정하는 과정에서 메뉴 아이디어 부족과 계획 부담을 느끼며, 이로 인해 요리를 미루거나 간편식·외식에 의존하는 경향이 있다. (UX 리서치, Allspice Labs 연구 등)
        
    - 냉장고 속 재료를 충분히 활용하지 못한다.
        - 국내 식폐기물은 연간 약 502만 톤으로, 전체 도시 rejet의 약 1/3 수준이며, 이는 냉장고/식량 관리에서 재료를 충분히 쓰지 못하고 남는 경우에서 많이 발생한다. https://www.khan.co.kr/en/article/202606121608007
        - 한국은 2023 년 기준 하루 약 1만 4천 톤의 음식물 쓰레기를 발생하며, 이는 전체 생활폐기물 발생량의 약 28.7%에 해당한다. https://superlocal.percent.ac/problems/food-waste
        - 2024 년 전국폐기물통계조사에서도 가정에서 발생하는 음식물 쓰레기 비율이 전체 식폐기물 중 상당 부분을 차지하며, 이 중 상당량이 냉장고·식량 관리 실패에서 기인하는 것으로 추정된다. https://mods.go.kr/board.es?mid=a10409070000&bid=12030&tag=&act=view&list_no=434431
        - 2022년 전세계적으로 1 인당 연간 식품 폐기량 132 kg이며, 이 중 60%가 가정에서 발생한다.https://www.foodicon.co.kr/news/articleView.html?idxno=25140
        
        ⇒  국내에서는 하루 약 1만 4천 톤의 음식물 쓰레기가 발생하며, 이 중 상당 부분이 가정에서 발생한다. 또한 환경부 조사에 따르면 수도권 가구는 평균 34종의 식재료를 보관하지만, 2주 동안 채소류와 냉동식품 등을 중심으로 상당량이 폐기되었으며, 냉장고 속 식재료를 잊어버리는 것이 주요 원인 중 하나로 나타났다.
        
    - 기존의 레시피 추천, 냉장고 관리 서비스는 문제를 충분히 해결하지 못한다.
        - Allspice Labs 연구에서도 레시피 앱/사이트 사용이 높은데도 “메뉴 아이디어 생성” 기능은 만족도가 낮고, 사용자가 원하는 것은 “간단한 조건(재료, 시간, 칼로리)으로 딱 맞는 메뉴 추천”인데 기존 서비스는 대부분 “검색+커스텀” 방식이라 부담이 크다고 분석된다. https://www.allspicelabs.com/data/study/
        - 현재 레시피 추천·냉장고 관리 앱은 대부분 “재료 검색 → 레시피 찾기” 방식에 머무르고, 입력한 재료를 기반으로 자동 추천해주는 기능이 미흡하거나, 재료 입력이 복잡해서 사용자가 “앱이 가치를 추가하지 못한다”거나 “앱이 제한적이다”라고 느끼는 경우가 많다. https://infor-basket5.tistory.com/1573
        - 여러 레시피 앱들을 비교한 글에서 공통적으로 언급된 문제: “재료 검색이 까다롭고, 재료 입력이 많으면 입력 부담이 크다.” “레시피 추천이 ‘과용량’이 많고, 한국형 입맛에 맞지 않는 경우가 많다.” “아주 일반적인 레시피만 나와서, 냉장고 재료를 딱 맞아떨어지게 추천하지 못한다.” https://mockingwalking.tistory.com/127?category=1198279
        
        ⇒ 기존 레시피 추천·냉장고 관리 서비스는 입력 부담이 크고, 제안이 일반적이라 사용자들이 “앱이 가치를 추가하지 못한다” 또는 “앱이 제한적이다”라고 느끼며 실제로 사용하지 않는 경우가 많다 (UX 리서치 사례, 기존 유사 서비스 VOC 분석)
        
    - 해결할 가치가 있는 문제다.
        - 환경부 등에 따르면, 한국은 하루 평균 약 1만 6천 톤의 음식물 쓰레기가 배출되며, 이는 연간 20조 원 이상의 경제적 손실과 8600 억 원의 처리 비용으로 이어진다. https://biz.heraldcorp.com/article/2129370
        - 한국농촌경제연구원 보고서에서, 한국 가정의 1 인당 연간 농식품 폐기량은 71 kg으로 OECD 평균(74.7 kg)보다 다소 낮지만, 식생활 문화가 비슷한 일본(64 kg), 중국(64 kg)보다 10% 이상 많다. 이는 “냉장고 재료를 충분히 활용하지 못해 버려지는 식재료”가 일상적 문제이며, 소비자의 경제적 부담도 실제 수준임을 시사한다. https://www.etoday.co.kr/news/view/2130782
        
        ⇒가정 내 식재료를 효율적으로 활용하지 못하는 문제는 개인의 불편을 넘어 사회적 비용과 환경 문제로 이어지는 규모 있는 문제이다.
        
- **문제 원인** : 사용자는 보유한 식재료를 기반으로 만들 수 있는 메뉴를 쉽게 찾거나 떠올리기 어렵다.
- **문제 정의** : 사용자는 보유한 식재료로 만들 수 있는 메뉴를 쉽게 떠올리기 어려워, 메뉴를 결정하는 데 불필요한 시간과 고민을 소비한다.

## 2. 핵심 기능 요구사항 (Requirements)

- **MVP(핵심 기능)** : 사용자의 식재료 상태를 기반으로 메뉴를 추천하는 시스템
- **기능 범위**
    1. **즉시 추천 모드(A)** : 사용자가 식재료를 직접 선택, 선택된 재료를 기반으로 즉시 메뉴 추천.
    2. **냉장고 상태 기반 추천 모드(B)** : 사용자가 저장된 냉장고 재료를 확인하고 추천에 사용할 재료를 선택하여 메뉴를 추천. 메뉴를 선택하고 요리를 완료하면 요리 기록이 저장된다.
    3. **냉장고 관리(C)** : 사용자가 냉장고 식재료를 조회·추가·수정·삭제하여 관리하는 기능. 관리된 냉장고 상태는 상태 기반 추천의 기반 데이터로 활용된다.
- **사용자 플로우** :
    - **전체 구조** : 서비스 진입 → 모드 선택 → 즉시 추천 / 냉장고 기반 추천 / 냉장고 관리 → 메뉴 추천 요청 → 추천 결과 확인(3개) → 메뉴 선택 → 레시피 상세 보기 → *(상태 기반 추천의 경우)* 요리 완료 → 요리 기록 저장
    - **A 플로우(즉시 추천)** : 서비스 진입 → 재료 선택 → 메뉴 추천 요청 → 추천 결과 확인(메뉴 3개의 사용된 재료, 예상 조리 시간, 간단 설명 포함) → 메뉴 선택 → 레시피 보기
    - **B 플로우(냉장고 기반 추천)** : 서비스 진입 → 내 냉장고 → 재료 확인 및 선택 → 메뉴 추천 요청 → 추천 결과 확인 → 메뉴 선택 → 레시피 보기 → 요리 완료 → 요리 기록 저장
    - **C 플로우(냉장고 관리)** : 서비스 진입 → 냉장고 관리 → 재료 조회 → 재료 추가/삭제/수정 → 저장
- **예외 플로우** :
    - 냉장고가 비어 있는 경우 : "냉장고가 비어 있어요." 메시지를 표시하고, "재료를 추가해주세요."와 냉장고 관리 화면으로 이동할 수 있도록 안내
    - 추천 가능한 메뉴가 없는 경우 : "해당 재료로는 추천 가능한 메뉴가 없어요." 메시지를 표시하고, "유사 재료를 포함해 추천받기" 또는 "재료를 추가해주세요."를 안내
    - 재추천 : “다른 메뉴 보기” 버튼 → 동일 조건으로 다른 결과 새로 생성
- **기능 요구사항 정의**
    - 재료 입력 기능 : 직접 선택해 메뉴 추천 받기
    - 냉장고 관리 기능 : 냉장고 식재료 조회·추가·수정·삭제
    - 메뉴 추천 엔진 : 재료 기반 매칭 추천, 3개 결과 제공
    - 상태 관리 기능 : 냉장고 재료를 저장·조회한다.
- **백엔드(Supabase) 연동** :
    - **냉장고 데이터 저장**: 사용자가 냉장고 관리 화면에서 입력한 식재료(식재료, 소스 등)는 유저별로 Supabase DB에 저장되어 이후 추천에 활용되어야 함.
    - **메뉴 추천 처리**: 사용자가 메뉴 추천을 요청하면 선택한 모드에 따라 입력한 재료 또는 저장된 냉장고 재료를 기반으로 메뉴를 추천해야 함.
    - **요리 완료 기록:** 사용자가 **요리 완료**를 선택하면 요리 기록을 저장한다.
    - **냉장고 상태 조회**: 사용자가 상태 기반 추천 모드에 진입하면 저장된 냉장고 재료를 불러와 현재 냉장고 상태와 메뉴 추천에 활용해야 함.
- **핵심 관계 정리 :**
    
    [ 냉장고 데이터 ] : 사용자 식재료 정보(DB)
    
    [ 상태 기반 추천 (Stateful Mode) ] : 냉장고 데이터를 기반으로 메뉴 추천
    
    [ 냉장고 관리 ] : 냉장고 상태를 조회·추가·수정·삭제하는 기능.
    

## 3. UI/UX 와이어프레임 및 디자인 가이드

- **화면 흐름도(Screen Flow + IA) :**



- **IA(Screen List)**
    
    
    | 화면 | 주요 기능 |
    | --- | --- |
    | 홈 | 추천 방식 선택 |
    | 재료 선택 | 식재료 직접 선택 |
    | 내 냉장고 | 저장된 재료 선택 |
    | 냉장고 관리 | 재료 조회·추가·수정·삭제 |
    | 추천 결과 | 메뉴 추천 결과 확인, 재추천 |
    | 레시피 | 레시피 확인 및 요리 완료 |
- **화면 설계 :**
    - **홈(Home)**
        - 목적 : 사용자가 원하는 메뉴 추천 방식을 선택하여 추천 프로세스를 시작하는 화면
        - 사용자 행동 : 서비스에 진입해 즉시 추천, 냉장고 기반 추천 또는 냉장고 관리 중 원하는 기능을 선택한다.
        - 화면 구성
            - 출력(Output)
                - 서비스 로고 또는 서비스명
                - 한 줄 소개 : *"냉장고 속 재료로 오늘의 메뉴를 추천해드립니다."*
                - **즉시 추천** 버튼
                - **냉장고 기반 추천** 버튼
                - **냉장고 관리** 버튼(텍스트 버튼)
            - 주요 액션
            
            | 액션 | 결과 |
            | --- | --- |
            | 즉시 추천 선택 | 재료 선택 화면으로 이동 |
            | 냉장고 기반 추천 선택 | 내 냉장고 화면으로 이동 |
            | 냉장고 관리 선택 | 냉장고 관리 화면으로 이동 |
            - 예외 처리 : 없음
        - 화면 와이어프레임
            - 상단 : 서비스 로고 / 서비스명. 그 아래 “냉장고 속 재료로 오늘의 메뉴를 추천해드립니다.” 카피
            - 중앙 ~ 하단 : 주요 버튼 3개
                - 즉시 추천 → “지금 생각나는 재료로 빠르게 추천”
                - 냉장고 기반 추천 → “내 냉장고 재료 기반으로 추천”
                - 냉장고 관리 (텍스트 버튼)
    - **재료 선택 화면**
        - 목적 : 사용자가 현재 보유한 식재료를 선택하여 해당 재료를 기반으로 메뉴를 추천받을 수 있도록 한다.
        - 사용자 행동 : 검색, 인기 재료 또는 카테고리 필터를 활용해 식재료를 선택한다. 검색어를 입력하면 인기 재료 영역이 검색 결과로 전환된다. 선택한 재료를 확인하거나 제거할 수 있다. 재료 선택이 완료되면 '메뉴 추천받기' 버튼을 눌러 메뉴 추천을 요청한다.
        - 화면 구성
            - 입력(Input)
                - 식재료 검색
                - 인기 재료 선택
                - 카테고리 필터 선택
                - 선택한 재료 삭제
            - 출력(Output)
                - 재료 검색창
                - 카테고리 필터
                - 재료 목록(기본: 인기 재료 / 검색, 필터 적용 시: 검색 결과)
                - 선택한 재료 목록(Chip 형태)
                - 메뉴 추천받기 버튼
            - 주요 액션(Action)
            
            | 액션 | 결과 |
            | --- | --- |
            | 재료 검색 | 검색 결과를 표시한다. |
            | 카테고리 선택 | 해당 카테고리의 재료만 필터링하여 표시한다. |
            | 인기 재료 선택 | 선택한 재료 목록에 추가된다. |
            | 검색 결과 선택 | 선택한 재료 목록에 추가된다. |
            | 선택한 재료 삭제 | 선택한 재료 목록에서 제거된다. |
            | 메뉴 추천받기 | 추천 결과 화면으로 이동한다. |
            - 예외 처리(Exception)
                - 선택한 재료가 없는 경우 **'재료를 1개 이상 선택해주세요.'** 메시지를 표시하고 메뉴 추천을 요청할 수 없다.
                - 추천 가능한 메뉴가 없는 경우 ‘**해당 재료로는 추천 가능한 메뉴가 없어요. 재료를 추가해주세요**'를 안내한다.
        - 화면 와이어프레임
            - 상단 : “재료 선택” 타이틀
            - 중앙 :
                - 식재료 검색창
                - 카테고리 필터
                - 인기 재료 리스트 (또는 검색 결과 리스트)
                - 선택 가능한 재료 리스트
            - 하단 :
                - 선택된 재료 chip 영역
                - **메뉴 추천받기 버튼 (Primary CTA)**
    - **내 냉장고 화면**
        - 목적 : 사용자가 저장된 냉장고 재료를 확인하고, 추천에 사용할 재료를 선택할 수 있도록 한다. 필요 시 냉장고 관리 화면으로 이동하여 재료를 관리할 수 있다.
        - 사용자 행동: 저장된 냉장고 재료를 확인한다. 추천에 사용할 재료를 선택하거나 선택 해제한다. 
        필요하면 **'냉장고 관리'** 버튼을 눌러 재료를 추가·삭제·수정한다. 재료 선택이 완료되면 '**메뉴 추천받기**' 버튼을 눌러 메뉴 추천을 요청한다.
        - 화면 구성
            - 입력(Input)
                - 추천에 사용할 재료 선택/선택 해제
                - 재료 검색
                - 냉장고 관리 화면으로 이동
            - 출력(Output)
                - 저장된 냉장고 재료 목록
                - 재료 검색창
                - 선택 상태 표시
                - **냉장고 관리** 버튼
                - **메뉴 추천받기** 버튼
            - 주요 액션(Action)
            
            | 액션 | 결과 |
            | --- | --- |
            | 재료 선택/선택 해제 | 추천에 사용할 재료가 반영된다. |
            | 재료 검색 | 저장된 재료를 검색한다. |
            | 냉장고 관리 | 냉장고 관리 화면으로 이동한다. |
            | 메뉴 추천받기 | 추천 결과 화면으로 이동한다. |
            - 예외 처리(Exception)
                - 저장된 재료가 없는 경우 **'냉장고가 비어 있어요.'** 메시지를 표시하고, **'재료를 추가해주세요.'**를 안내한다.
                - ‘**냉장고 관리**’ 버튼을 눌러 재료를 추가할 수 있도록 한다.
                - 추천에 사용할 재료를 선택하지 않은 경우 **'재료를 1개 이상 선택해주세요.'** 메시지를 표시하고, ‘**메뉴 추천받기**’ 버튼은 비활성화 된다.
        - 화면 와이어프레임
            - 상단 : “내 냉장고” 타이틀. 냉장고 상태 요약(예: “총 12개 재료 보관 중”)
            - 중앙 :
                - 저장된 재료 리스트(재료명, 수량 / 상태 표시 (옵션))
                - 선택된 재료 highlight 상태
                - 재료 검색창 (상단 또는 중앙 상단 배치)
            - 하단 :
                - 냉장고 관리 버튼
                - 메뉴 추천받기 버튼 (재료가 선택된 경우 활성화)
    - **냉장고 관리 화면**
        - 목적 : 사용자가 냉장고에 있는 식재료를 직접 추가·수정·삭제하며 냉장고 상태를 관리할 수 있도록 한다. 이 데이터는 이후 메뉴 추천의 기준이 된다.
        - 사용자 행동: 사용자는 현재 냉장고에 저장된 재료를 확인하고, 필요에 따라 재료를 추가하거나 삭제·수정한다. 재료는 검색 또는 직접 입력으로 추가할 수 있으며, 유통기한 등 기본 정보도 함께 관리할 수 있다. 관리가 완료되면 홈 또는 이전 화면으로 돌아간다.
        - 화면 구성
            - 입력(Input)
                - 식재료 검색
                - 식재료 직접 추가
                - 기존 재료 수정 (수량/유통기한 등)
                - 재료 삭제
            - 출력(Output)
                - 냉장고 재료 리스트
                - 재료 상세 정보 (이름, 수량, 선택적으로 유통기한)
                - 재료 추가 버튼
                - 변경사항 저장 버튼
            - 주요 액션(Action)
            
            | 액션 | 결과 |
            | --- | --- |
            | 재료 추가 | 냉장고 리스트에 새로운 재료가 추가된다 |
            | 재료 수정 | 선택한 재료 정보가 업데이트된다 |
            | 재료 삭제 | 해당 재료가 냉장고에서 제거된다 |
            | 변경사항 저장 | 변경사항이 DB에 저장되고 이전 화면으로 이동 |
            - 예외 처리(Exception)
                - 동일 재료가 존재하는 경우 → “이미 존재하는 재료입니다. 수량을 수정해주세요.”
                - 입력값이 비어 있는 경우 → “재료 이름을 입력해주세요.”
                - 저장 실패 시 → “냉장고 정보 저장에 실패했습니다. 다시 시도해주세요.”
        - 화면 와이어프레임
            - 상단: 냉장고 타이틀 + 검색창
            - 중앙: 재료 리스트 (카드 또는 리스트 형태)
            - 하단: + 재료 추가 버튼 / 저장 버튼
    - **추천 결과 화면**
        - 목적 : 즉시 추천 모드에서는 사용자가 선택한 재료를, 냉장고 상태 기반 추천 모드에서는 냉장고 데이터를 기반으로 추천된 메뉴 리스트를 제공하고, 최종 메뉴 선택으로 이어지도록 한다.
        - 사용자 행동: 사용자는 추천된 메뉴 리스트(3개)를 확인하고, 각 메뉴의 간단 정보(사용 재료, 예상 조리 시간, 설명)를 비교한다. **마음에 드는 메뉴를 선택**하거나 **다른 메뉴 보기**를 통해 재추천을 요청할 수 있다.
        - 화면 구성
            - 입력(Input)
                - **다른 메뉴 보기** 요청
                - 메뉴 선택
            - 출력(Output)
                - 추천 메뉴 리스트 (3개)
                - 각 메뉴 카드 정보:
                    - 메뉴명
                    - 사용 재료 요약
                    - 예상 조리 시간
                    - 간단 설명
                - “다른 메뉴 보기” 버튼
            - 주요 액션(Action)
            
            | 액션 | 결과 |
            | --- | --- |
            | 메뉴 선택 | 레시피 화면으로 이동 |
            | 다른 메뉴 보기 | 동일 조건으로 새로운 추천 결과 생성 |
            - 예외 처리(Exception)
                - 추천 결과 없음 → “해당 조건으로 추천 가능한 메뉴가 없습니다.” 메시지와 함께 “재료 다시 선택하기” 버튼을 출력하기.
                - 추천 로딩 실패 → “추천 결과를 불러오지 못했습니다. 다시 시도해주세요.”
        - 화면 와이어프레임
            - 상단: “추천 결과” 타이틀 + 선택 재료 요약
            - 중앙: 메뉴 카드 리스트 (3개)
            - 하단: “다른 메뉴 보기” 버튼
    - **레시피 상세 화면**
        - 목적 : 선택한 메뉴의 상세 레시피를 제공하고, 사용자가 실제 요리를 진행할 수 있도록 돕는다.
        - 사용자 행동: 사용자는 선택한 메뉴의 레시피를 단계별로 확인하며 요리를 진행한다. 상태 추천 모드의 경우 요리가 완료되면 **요리 완료** 버튼을 눌러 요리 기록을 저장한다. 필요 시 이전 화면으로 돌아간다.
        - 화면 구성
            - 입력(Input)
                - 요리 완료 버튼
            - 출력(Output)
                - 메뉴명
                - 레시피 설명 (단계별 or 텍스트)
                - 필요 재료 목록
                - 조리 시간
                - **요리 완료** 버튼
            - 주요 액션(Action)
            
            | 액션 | 결과 |
            | --- | --- |
            | 요리 완료 | 요리 기록을 저장함. |
            | 뒤로가기 | 추천 결과 화면으로 이동 |
            - 예외 처리(Exception)
                - 레시피 데이터 없음 → “레시피 정보를 불러올 수 없습니다.”
                - 요리 기록 저장 실패 → “요리 기록 저장에 실패했습니다. 다시 시도해주세요.”
        - 화면 와이어프레임
            - 상단: 메뉴명 + 조리 시간
            - 중앙: 레시피 단계 또는 설명
            - 하단: 필요 재료 + **요리 완료** 버튼
- 종합 와이어프레임 :

- 디자인 원칙 :
    1. **“생각하지 않아도 되게 만든다”**
        1. 사용자가 메뉴를 고르는 과정을 최대한 단순화한다.
        2. 검색, 입력, 선택의 부담을 줄이고 “누르면 된다” 수준으로 만든다.
    2. **“냉장고 기반 추천이 목적이다”**
        1. 즉시 추천은 있지만, 우리의 메인 목표는 냉장고 기반 추천이다.
        2. 냉장고 기반 추천을 기점으로 우리 서비스로 냉장고를 관리하게 만든다. 다만, 관리되고 있다는 경험을 줘야 한다.
    3. **“결정은 빠르게, 선택은 가볍게”**
        1. 메뉴 고민 시간을 줄이고, 3개의 선택지로 의사결정을 쉽게 만든다.
    4. **“관리해야 하는 것이 아닌, 관리되고 있는 경험”**
        1. 사용자가 요리하면 요리 기록이 저장된다.
        2. 사용자가 관리하는 느낌이 아니라 “관리되고 있는 경험”을 제공한다.
    5. **“모든 흐름은 홈으로 돌아온다”**
        1. 추천 → 선택 → 요리 → 종료가 아니라 항상 다시 “뭐 해먹지?”로 돌아오는 순환 구조를 만든다.

## 4. 메뉴 추천 로직

- 추천 흐름
    - 사용자가 선택한 재료 수집
    - 저장된 냉장고 데이터를 사용하거나 사용자가 선택한 재료를 사용한다
    - 재료 기반으로 후보 레시피 필터링
    - 조건에 맞는 메뉴 3개 추출
    - 결과 반환
- 추천 기준(위에서부터 중요)
    - 필수 재료 충족 여부(가장 중요)
        - 핵심 재료가 없는 메뉴는 제외
    - 재료 일치율
        - 사용자가 가진 재료와 많이 겹치는 메뉴 우선
    - 부족 재료 최소화
        - 추가로 필요한 재료가 적을수록 우선
    - 조리 시간
        - 짧은 시간 우선 (옵션 가중치)
- 재추천 로직(위에서부터 중요)
    - 동일 조건으로 다시 3개 재샘플링
    - 이미 보여준 메뉴는 제외
    - 후보가 1개 이상 3개 미만인 경우에는 조건 완화:
        - 유사 재료 허용 (예: 닭가슴살 ↔ 닭고기)
        - 일부 필수 재료 제외 기준 완화
- 예외 처리
    - 추천 가능한 메뉴 없음:
        - “해당 재료로 만들 수 있는 메뉴가 없습니다.”
        - 처리 : 재료 추가 유도 또는 조건 완화 추천(유사 재료 포함)
    - 선택 재료 없음 : 추천 버튼 비활성화 + “재료를 선택해주세요” 메시지

## 5. 데이터 및 백엔드 설계

- DB 구조
    - users(유저)
        - id
        - created_at
    - fridge(냉장고 재료)
        - id
        - user_id
        - name (재료명)
        - quantity (수량, optional)
        - updated_at
    - recipes (레시피)
        - id
        - name
        - cooking_time
        - description
    - recipe_ingredients (레시피 재료)
        - recipe_id
        - ingredient_name
        - is_required (true/false)
    - cooking_history (요리 기록)
        - id
        - user_id
        - recipe_id
        - created_at
- 데이터 관계
    - user 1 → N fridge
    - user 1 → N cooking_history
    - 추천은 fridge 데이터를 기반으로 recipes를 조회
    - recipe 선택 시 cooking_history에 저장
- API 구조
    - 냉장고 관련
        - GET /fridge → 냉장고 조회
        - POST /fridge → 재료 추가
        - DELETE /fridge/:id → 삭제
        - PATCH /fridge/:id → 수정
    - 추천 관련
        - POST /recommend
            - input: 추천에 사용할 재료 리스트
            - output: 추천 메뉴 3개
    - 레시피
        - GET /recipes/:id
- 데이터 흐름
    - 즉시 추천 : 사용자 선택 재료 → recommend API → 결과 반환
    - 냉장고 기반 추천 : fridge 조회 → 선택 → recommend API → 결과 반환
    - 요리 완료 : recipe 선택 → 요리 기록 저장
- 상태 개념
    - 냉장고 데이터 = “source of truth”
    - 추천 = “읽기 전용 결과”
